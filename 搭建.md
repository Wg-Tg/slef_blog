## 一、项目初始化 & 技术栈约束

```
请帮我初始化一个 Astro 项目，并集成 Vue 3、UnoCSS 和 Three.js，用于构建一个“太阳系导航”风格的个人网站。

必须遵循的技术栈：
- Astro（JAMStack，默认静态生成）
- Vue 3（用于交互岛屿组件，通过 @astrojs/vue）
- UnoCSS（原子化 CSS，按需生成）
- GSAP（高级动效）
- Three.js（3D 场景，可通过 TresJS 或原生集成）

严格禁止：
- 蓝紫渐变、毛玻璃（backdrop-filter blur）、霓虹光效
- 过于圆润的大圆角（>8px）
- 复杂的服务端框架或数据库

请执行以下步骤：
1. 使用 `npm create astro@latest` 初始化，选择：
   - 包含示例文件
   - 启用 Vue 框架集成
   - 启用 View Transitions 实验功能（experimental: { viewTransitions: true }）
   - TypeScript 严格模式
2. 安装并配置 UnoCSS：
   ```bash
   npm install -D unocss @unocss/astro
```
   在 `astro.config.mjs` 中引入 `@unocss/astro`。
3. 创建 `uno.config.ts`，使用 `presetWind` 和 `presetTypography`，并预留自定义主题扩展。
4. 安装 GSAP：`npm install gsap`
5. 安装 Three.js 相关（选择一种方式）：
   - 若使用 TresJS：`npm install three @tresjs/core @tresjs/cientos`
   - 若原生 Three.js：`npm install three @types/three`
6. 确保项目结构包含 `src/content/` 用于 Markdown 内容，`src/pages/` 用于路由。
7. 构建输出目录为 `dist`，添加脚本 `"deploy": "astro build"`。



## 二、设计系统与主题配置

现在配置整个网站的设计系统。风格定位：新粗野主义 × 纸媒体质感，带有星际暗色调点缀。

请修改 `uno.config.ts` 和全局样式，定义如下令牌，并生成 `src/styles/global.css`。

颜色：
- 主背景：米白 `#FBF9F6`（用于内容页）
- 表面：纯白 `#FFFFFF`
- 主文字：墨黑 `#1A1A1A`
- 强调色：朱红 `#E63946`
- 辅助强调：芥末黄 `#F4A261`
- 边框：深灰 `#2B2B2B`
- 柔和灰：`#6B6B6B`
- 太空黑：`#0B0A0A`（太阳系首页背景）
- 星点白：`#EAEAEA`

字体：
- 正文：`Georgia` 或 `'Times New Roman'`（衬线）
- 标题：`'DM Sans'` 或系统无衬线，粗体，字间距 `-0.02em`
- 等宽：`'JetBrains Mono'` 用于代码块

边框与阴影：
- 全局边框：`2px solid #2B2B2B`
- 卡片/容器默认阴影：`box-shadow: 6px 6px 0px #2B2B2B`（硬阴影）
- 按钮悬浮时阴影缩小为 `2px 2px 0px #2B2B2B`，产生按压感
- 元素圆角最大 `8px`，大部分使用 `0` 或 `4px`

排版：
- 最大内容宽度 `720px`，居中
- 标题 h1 `3.5rem`，行高 `1.1`
- 段落行高 `1.8`，字间距 `0.01em`
- 代码块背景 `#F5F5F5`，无阴影

全局 CSS 要求：
- 应用主背景、字体
- 定义 `::view-transition-old` 和 `::view-transition-new` 的默认动画（后文会细化）
- 将所有颜色定义为 CSS 变量，供 UnoCSS 和普通 CSS 共用

3. 请生成 `src/content/config.ts` 和相应的示例 Markdown 文件（博客至少 2 篇，碎片至少 2 条），确保类型安全。

## 五、太阳系导航首页（核心交互入口）

现在我们要把根路由 `/` 变成一个可交互的 3D 太阳系。这是全站的入口，点击行星将平滑过渡到对应页面。

1. 创建 Vue 组件 `src/components/SolarSystem.vue`（仅客户端渲染 `client:only="vue"`）。
2. 使用 Three.js 初始化场景（通过 TresJS 或原生），遵循以下视觉规范：
   - 背景：深邃太空色 `#0B0A0A` 到 `#1A1A2E` 的微渐变（CSS 背景），Three.js 场景透明背景
   - 太阳：居中，暖金色球体，带有微弱的 emissive 光，无后期泛光；可添加一个缓慢旋转的粒子环
   - 行星与映射（点击跳转）：
     - 水星（小，铁灰色） → `/blog`
     - 金星（中等，暖橙色） → `/notes`
     - 地球（蓝绿色程序化纹理） → `/hobbies`
     - 火星（红色调） → `/projects`（如果未建，可先指向 `/about`，稍后调整）
     - 木星（最大，浅棕条纹） → `/about`
     - 土星（带环，淡黄色） → `/contact`（暂缺，可指向 `/` 或稍后创建）
   - 每个行星有对应的圆形轨道线（半透明白色，线宽 1px）
   - 背景散布 600-800 个随机星点（移动端 300 个）
3. 交互：
   - 鼠标悬停行星：行星略微放大（1.15 倍），光标变为 pointer，并显示一个 HTML 浮动标签（米白底、硬边框、黑色文字，内容为行星名与中文说明，如“水星 · 博文”）
   - 点击行星：不直接使用 `location.href`，而是通过自定义事件通知 Astro 页面进行导航，以保留 View Transitions。实现方法：
     - 在 `SolarSystem.vue` 中 dispatch 事件：`document.dispatchEvent(new CustomEvent('planet-click', { detail: { url: '/blog' } }))`
     - 在 `src/pages/index.astro` 的 `<script>` 中监听该事件并调用 Astro 的 `navigate(url)` 函数
   - 相机：允许鼠标拖拽旋转（OrbitControls），限制缩放范围；移动端支持触摸旋转
   - 性能：`requestAnimationFrame` 驱动，卸载时清理资源；考虑 `prefers-reduced-motion` 时降低动画速度或禁用旋转
4. 首页 `index.astro`：
   - 只渲染 `<SolarSystem client:only="vue" />` 和一个监听导航事件的脚本
   - 隐藏导航栏（或让导航栏透明悬浮），使太阳系充满全屏
   - 添加 CSS 使页面背景为太空黑

5. 视图过渡衔接：
   - 在全局 CSS 中，针对离开太阳系时添加 `::view-transition-old(root)` 的动画：缩小至 `0.9`、旋转 `10deg`、淡出，营造“飞入行星”的感觉。

请生成完整的 `SolarSystem.vue`、修改后的 `index.astro` 以及相关 CSS 和事件处理代码，确保无刷新平滑跳转。

## 六、博客与碎片流（核心内容页面）

现在创建博客列表、详情和碎片分享页面。

   - 使用 `getStaticPaths` 和 `getEntry` 动态渲染
   - 正文使用 UnoCSS prose 排版，图片自动添加 `2px` 黑边框和硬阴影
   - 底部预留评论区 `<div id="comments">`（后续集成 Giscus）

3. 碎片流页面 `src/pages/notes/index.astro`：

     - 提取所有唯一标签，渲染为切换按钮

     - 使用 `<TransitionGroup>` 配合 GSAP FLIP 动画，实现卡片过滤时的平滑增删位移
   - 无结果时显示硬阴影风格的空状态提示：“还没有这样的碎片。”
   - 确保服务端渲染初始列表，客户端仅增强过滤。

4. 创建 `<ScrollReveal>` Vue 组件：
   - 使用 GSAP ScrollTrigger，包裹默认插槽
   - 元素进入视口时从下方 40px 淡入，duration 0.6s，Power3.out，只触发一次

5. 更新导航栏链接指向正确路径。
